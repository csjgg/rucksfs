# 论文写作 Agent 规范

本文件约束所有参与论文写作和修改的 Agent 的行为。

## 1. 反 AI 检测与语言规范性的平衡（核心原则）

**反 AI 检测和学术写作规范是需要平衡的两个目标，不可为了其中一个牺牲另一个。**

### 优先级排序

1. **语义准确、逻辑清晰** — 最高优先级。读者能顺畅理解是底线。
2. **学术中文规范** — 句式完整、用词准确、段落结构合理。
3. **反 AI 检测** — 在前两条满足的前提下，避免 AI 特征模式。

### 具体原则

- **不为反检测而制造混乱**：如果一个句子用"第一…第二…第三…"结构最清晰，就用这个结构。不要为了"打破对称"而强行打乱逻辑顺序。
- **不用口语替代学术表述**：反 AI 不等于口语化。"这个问题不好办"不如"该问题的工程复杂度较高"。
- **自然参差优于刻意参差**：句式长短交替应该来自内容本身的需要（有的概念一句话能说清，有的需要长句展开），而非人为制造。
- **作者判断要有依据**：直说"过度设计"可以，但必须跟上理由（"对于中小规模场景，Raft 共识的延迟开销超过了其带来的容错收益"），不能只有态度没有论据。

### 反 AI 检测规则（在规范性前提下遵守）

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | **砍模板化开头** | 禁止"随着...的不断发展""近年来X的快速发展带来了Y的Z"式开头。用具体事实、数据或问题直接切入。 |
| R2 | **避免高度对称的句式** | 不宜连续三段以上使用完全相同的句式骨架。但如果并列结构是最清晰的表达方式（如列举三个技术挑战），允许保留。 |
| R3 | **减少路标式过渡** | 尽量不用"本章首先…随后…最后…"。但如果段落间逻辑跳跃较大，一句简短的过渡是允许的。 |
| R4 | **加入作者判断** | 该说"不足"就直说，但必须附带依据。避免无依据的主观评价和永远中立的罗列式写法。 |
| R5 | **减少泛化断言** | 避免"已被广泛应用""具有重要意义"等空话。要么删掉，要么换成带数据或引用的具体说法。 |
| R6 | **自然的语气变化** | 在严谨表述中可以穿插简短判断或直接转折，但前提是不破坏语义连贯性。 |

### AI 高频词黑名单（优先替换）

- "进一步" → 删掉，或用"在此基础上"
- "从根源上" → 换为具体描述
- "表明" → 交替使用"说明""印证""可以看出""数据显示"
- "值得注意的是" → 直接陈述
- "不可忽视" → 用数据说话
- "与此同时" → 删掉或换为"此外""另一方面"
- "彻底" → 通常删掉

## 2. 专业性与行文规范

- **学术中文**：全文使用规范学术中文，避免口语化、营销化表述
- **术语一致**：同一概念全文使用同一译名和缩写，首次出现给全称
- **符号一致**：同一变量全文使用同一符号
- **段落控制**：单段不超过 200 字（约 10 行），超过则拆分；单句不超过 50 字，超过则断句
- **过渡自然**：章节之间用内容逻辑衔接，不用"下面将介绍..."式导游词
- **标点规范**：正文中文标点，公式和代码内英文标点。数字与单位间加 `\,`（如 `64\,MB`）

## 3. 引用与数据真实性（红线）

- **所有引用的论文必须真实存在**：不允许编造论文标题、作者、会议名或年份。如果不确定某篇论文是否存在，宁可不引用。
- **所有实验数据必须来自实际测试结果**：不允许编造性能数据。摘要和正文中的 ops/s、倍数、百分比必须与实验章节的表格一致。
- **引用数量要求**：全文引用论文不少于 25 篇（当前 ref.bib 有 39 条，足够）。
- **数据交叉验证**：同一数据在摘要、正文、表格中出现时必须一致。TiKV 对比倍数为 1.7（9917/5908），不是 1.5。

## 4. LaTeX 规范

- 表格使用 `booktabs` 三线表，标题在表上方
- 图片标题在图下方
- 引用格式：`图~\ref{fig:xxx}`、`表~\ref{tab:xxx}`、`第~\ref{sec:xxx}~节`
- 算法使用 `algorithm` + `algorithmic` 环境
- 代码片段使用 `verbatim` 或 `minted`（如已配置）

## 5. 文件操作规范

- 修改 `.tex` 文件前必须先读取当前内容
- 每次修改使用 Edit 工具做定向替换，不做全文重写（除非用户明确要求）
- 修改后确认文件内容正确
- 中英文摘要保持同步
